# Prototype notes

This branch intentionally implements the complete core flow in one reviewable prototype rather than splitting it into separate feature phases.

Device validation priorities:

1. Add an ingredient and restart the app.
2. Build a recipe with multiple ingredients.
3. Log one serving and half a recipe.
4. Confirm the dashboard changes immediately.
5. Delete a meal from History.
6. Generate a grocery list from multiple recipes.
7. Run unit tests and report any Gradle or Room migration errors.
